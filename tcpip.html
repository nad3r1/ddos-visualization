Concept Map

TCP/IP Model and DDoS Mapping

This page explains how the project uses the TCP/IP model to organize attacks, protocols, and security relevance.

Explanation

How The Model Is Used In This Project

Application Layer

This layer includes application protocols such as HTTP and DNS. In this project, HTTP Flood and DNS Amplification both depend on application-level protocol behavior, even though their damage can spread into lower layers.

Transport Layer

This layer includes TCP and UDP. SYN Flood attacks TCP connection setup, while UDP Flood attacks packet delivery capacity without needing a connection handshake.

Internet and Network Access Effects

Some attacks begin with one protocol but create wider path-level effects. DNS Amplification especially shows how source spoofing and reflected traffic can overwhelm upstream routing paths and access links.

Layer Map

Layer, Protocol, and Security Relevance

A DDoS attack may technically target one layer while creating practical effects across several layers. That is why the model is useful as a teaching structure rather than as a rigid boundary.

Interpretation

Layer vs. Impact

DNS Amplification Across Layers

DNS Amplification is a useful example because its behavior begins at the Application layer with DNS queries and responses, travels over UDP at the Transport layer, depends on spoofed source IP addresses at the Internet layer, and can ultimately congest routing paths and access links when the reflected response volume becomes large enough.

SYN Flood As A Contrast

SYN Flood is a simpler contrast because it directly abuses TCP connection setup at the Transport layer. Even so, the effect is still experienced by applications and users because sessions cannot be established reliably once connection state is exhausted.
