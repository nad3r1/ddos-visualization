Project Overview

Why This Topic Matters

What DDoS Attacks Are

A Distributed Denial-of-Service attack uses many traffic sources to overwhelm a target with connection attempts, packets, or application requests. The goal is not usually to read or alter stored data directly, but to make the service slow, unreliable, or unreachable.

Why Availability Is The Main Security Property

In this project, the key security property under attack is availability. A service can keep its data confidential and unmodified, but still fail its users if they cannot connect or receive useful responses in time.

Why The TCP/IP Model Matters

DDoS attacks do not all behave at the same layer. Some target TCP connection setup at the Transport layer, some target application requests at the Application layer, and some create congestion that spreads across Internet and access paths. Mapping attacks to the TCP/IP model helps explain why defensive strategies must be chosen carefully.

How To Use This Site

Suggested Learning Path

1. Start With The Simulator

Open the simulator and compare one attack against multiple mitigations. This gives a fast visual sense of how different defensive controls change server load, attack pressure reduced, and service availability.

2. Read The Attack Guides

Use the dedicated attack pages to understand the protocol behavior, step-by-step attack process, and the specific bottleneck each attack exploits.

3. Read The Mitigation Guides

Review the mitigation pages to see why no single defense solves every DDoS problem and why layered security is a recurring theme in network defense.

Learning Goals

Learning Objectives

Availability Under Stress

Explain how DDoS attacks reduce service availability by exhausting connection state, server capacity, bandwidth, or request-processing resources before legitimate users can be served reliably.

Layer Identification

Identify which TCP/IP layers are most directly involved in SYN Flood, HTTP Flood, UDP Flood, and DNS Amplification, while also recognizing that practical service impact can cross layer boundaries.

Defense Fit

Describe why different DDoS mitigations work better for different attack types and why network security teams usually combine several controls rather than relying on a single defense.

Metric Interpretation

Interpret simplified simulator metrics such as server load, legitimate traffic served, attack pressure reduced, and service availability as relative teaching indicators rather than as real monitoring data.

Academic Framing

Research Basis

Attack Behavior

The project content is based on research into general DDoS behavior, TCP three-way handshake abuse in SYN floods, HTTP application-layer request flooding, UDP volumetric flooding, and DNS reflection/amplification. These attack families were selected because they demonstrate different pressure points across the TCP/IP model.

Mitigation Strategy

The mitigation guidance is grounded in commonly discussed defensive approaches such as rate limiting, SYN cookies, traffic filtering, load balancing, and CDN or Anycast absorption. The website uses these controls to show that mitigation quality depends on mechanism fit, not just on whether a tool sounds generally useful.

Performance Concepts

The simulator also draws on basic network performance ideas such as server load, traffic volume, reduced hostile pressure, and availability. The numbers shown in the simulator are deliberately simplified so the cause-and-effect relationship stays understandable in a classroom discussion.

Operational Context

Real-World Relevance

What DDoS Can Disrupt

DDoS attacks can affect public websites, APIs, DNS providers, gaming platforms, cloud services, and campus systems such as learning portals, authentication services, or university-facing web applications.

Why Security Teams Care

Security and network operations teams care about DDoS resilience because availability failures can interrupt learning, commerce, communication, and trust. Even when no data is stolen, a service that cannot be reached still fails its mission.
